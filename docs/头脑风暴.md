# AI Agent 软件自动化测试 — 项目与资料汇总

## 一、浏览器/Web 端到端测试 Agent

| 项目 | Stars | 核心思路 |
|---|---|---|
| **browser-use/browser-use** | ⭐97.7k | 最火的开源浏览器 Agent。LLM 直接操控浏览器，自动执行任务、填写表单、抓数据。Playwright 驱动 + 视觉理解 |
| **browserbase/stagehand** | ⭐23k | Browserbase 出品的浏览器 Agent SDK。`act()` / `extract()` / `observe()` 三个原语，AI 自动定位元素并操作 |
| **Skyvern-AI/skyvern** | ⭐21.9k | 自动化浏览器工作流。LLM + 视觉理解页面结构，能处理复杂多步业务流程 |
| **nanobrowser/nanobrowser** | ⭐13.1k | Chrome 扩展形态的 AI 自动化代理，多 Agent 协作，自带 LLM API key 即可用 |
| **web-infra-dev/midscene** | ⭐13.6k | 字节开源，**专为测试设计**。视觉驱动 UI 自动化，支持 Playwright / Puppeteer / 原生 App |

## 二、MCP 协议类（LLM ↔ 测试工具桥接）🆕

| 项目 | Stars | 核心思路 |
|---|---|---|
| **microsoft/playwright-mcp** | ⭐33.6k | 微软官方。把 Playwright 封装成 MCP Server，任何 LLM Agent 都能直接调用浏览器操作 |

> **MCP = Model Context Protocol**，Anthropic 提出的标准，让 LLM 能统一调用外部工具。playwright-mcp 是目前测试领域最重要的基础设施。

## 六、推荐入门路径

```
第一层：先用起来
  → midscene.js （字节，中文友好，专做测试）
  → playwright-mcp （微软，标准化接口）

第二层：看懂原理
  → browser-use （看源码，理解 Agent 如何"看"页面和"点"按钮）
  → omniparser （理解屏幕解析）

第三层：自己搭
  → langgraph / agno （编排多个 Agent 协作）
  → 自己的测试 Agent = LLM + MCP tools + 断言引擎
```
